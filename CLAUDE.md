# כלל חובה לפני כל שינוי קוד

לפני כל תיקון, שינוי או תוספת, חובה לעצור ולשאול:

1. **מה יכול להישבר?** - רשום לפחות 2-3 תרחישים כשלון אפשריים
2. **מה כבר עובד ועלול להיפגע?** - קרא את הקוד הקיים ובדוק שהשינוי לא שובר פונקציונליות קיימת
3. **האם יש תלויות?** - תוכנה פתוחה, תהליך רץ, API חיצוני
4. **איך אני מאמת?** - לא שהקוד רץ, אלא שהתוצאה האמיתית נכונה

רק אחרי מענה על כל השאלות - כותבים קוד.
