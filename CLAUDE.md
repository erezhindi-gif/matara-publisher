# כלל חובה לפני כל שינוי קוד

לפני כל תיקון או שינוי, חובה לעצור ולשאול:

1. **מה יכול להישבר?** - רשום לפחות 2-3 תרחישים כשלון אפשריים
2. **האם יש תלויות?** - תוכנה פתוחה, תהליך רץ, API חיצוני
3. **איך אני מאמת?** - לא שהקוד רץ, אלא שהתוצאה האמיתית נכונה

רק אחרי מענה על שלוש השאלות - כותבים קוד.
